# PRM - 个人关系管理工具

一个基于终端的 CLI 工具，帮助你系统化地维护人际关系，根据联系人等级、生活模式、地理位置自动计算联系周期，并提供智能话题推荐。

## 核心特性

- **分级管理**：S/A/B/C/D 五个等级，自动计算联系间隔
- **智能调度**：根据生活模式（实习/校园）动态调整联系频率
- **地理感知**：区分本地/外地联系人，线上联系间隔自动缩短
- **话题推荐**：记录历史话题和兴趣，联系时自动推荐
- **反馈学习**：根据聊天质量动态调整联系周期
- **休眠模式**：长期离线时冻结所有倒计时

## 快速开始

```bash
python3 prm.py
```

首次运行会自动加载 5 条演示数据。

## 联系人等级

| 等级 | 默认间隔 | 适用对象 |
|------|---------|---------|
| S    | 不调度   | 导师、家人等特别关注对象 |
| A    | 7 天    | 核心人脉、密切合作伙伴 |
| B    | 14 天   | 重要同学、同事 |
| C    | 30 天   | 一般联系人 |
| D    | 60 天   | 老朋友、弱联系 |

## 主要命令

### 查看与添加

- `view` - 查看本周日程视图
- `add` - 添加新联系人
- `list` - 列出所有联系人

### 记录与编辑

- `contact` - 记录一次联系（含话题推荐）
- `edit` - 编辑联系人信息
- `delete` - 删除联系人

### 系统设置

- `mode` - 切换生活模式（internship/campus）
- `city` - 设置本地城市
- `interval` - 调整等级联系间隔
- `pause` - 全局休眠/恢复

## 使用场景

### 1. 添加联系人

```
PRM> add
  姓名: 张三
  等级 (S/A/B/C/D): A
  标签 (逗号分隔，可留空): 实验室,向上社交
  常住地 [本地]: 北京
  联系方式 (线上/线下，逗号分隔) [线上,线下]: 线上
  兴趣爱好 (逗号分隔，可留空): 机器学习,摄影
```

### 2. 记录联系

```
PRM> contact
  联系人姓名: 张三
  💡 话题推荐:
     兴趣: 机器学习, 摄影
     最近话题: 论文进展, 实习经验
  反馈质量:
    1. 聊得很好
    2. 一般
    3. 没话找话
  请选择 (1/2/3): 1
  简短备注 (可回车跳过): 讨论了新项目
  聊天话题 (可回车跳过): AI应用落地
```

### 3. 切换生活模式

```
PRM> mode
  当前模式: campus
  可选模式: internship, campus
  切换到: internship
```

实习模式下，带"实验室"标签的联系人间隔会自动延长 2 倍。

## 智能特性

### 地理位置感知

- **本地联系人**：显示 `[线下]`，使用标准间隔
- **外地联系人**：显示 `[线上]`，间隔自动缩短至 0.7 倍

### 动态调整

每次联系后根据反馈质量调整：
- 聊得很好：周期缩短 10%
- 一般：保持不变
- 没话找话：周期延长 10%

### 模式乘数

在 `MODE_MULTIPLIERS` 中配置：
```python
MODE_MULTIPLIERS = {
    "internship": {"实验室": 2.0},  # 实习时实验室联系人间隔 x2
    "campus":     {"实验室": 0.5},  # 校园时实验室联系人间隔 x0.5
}
```

## 数据文件

- `prm_contacts.json` - 联系人数据
- `prm_config.json` - 系统配置

数据以 JSON 格式存储在脚本同目录，可手动备份。

## 高级用法

### 自定义等级间隔

```
PRM> interval
  当前各等级联系间隔 (天):
    S: 不参与调度
    A: 7 天
    B: 14 天
    C: 30 天
    D: 60 天
  输入要修改的等级 (A/B/C/D)，回车取消:
  等级: A
  A 级新间隔天数 [7]: 5
```

### 休眠模式

长期离线前启用休眠，恢复时所有联系人日期自动顺延：

```
PRM> pause
  [OK] 系统已进入休眠，所有倒计时冻结。

# 一周后恢复
PRM> pause
  [OK] 系统已恢复，休眠 7 天，所有联系人日期已顺延。
```

## 技术架构

- **Contact** - 联系人数据模型
- **Config** - 全局配置管理
- **ScheduleEngine** - 调度与状态判定引擎
- **ContactManager** - 业务操作层
- **CLIView** - 终端视图渲染
- **DataStore** - JSON 持久化层
